# 오늘의신기템 온라인 관리자 설계

날짜: 2026-07-12  
상태: 사용자 승인됨

## 목적

Netlify 온라인 관리자에서 상품, 콘텐츠 작업, 완성 영상, Instagram 게시, 광고와 성과를 통합 관리한다. 관리자는 한 명이며 외부에서도 접속한다. 영상 처리 PC는 필요할 때만 켜므로 작업 요청은 온라인 큐에 보존하고, 로컬 Worker가 접속할 때 처리한다.

## 확정 아키텍처

- Netlify: 온라인 관리자 프런트엔드와 기존 공개 링크 페이지
- Supabase Auth: 지정된 관리자 이메일 한 개만 허용
- Supabase Postgres: 상품, 콘텐츠, 작업 큐, 게시, 광고, 성과, 활동 기록
- Supabase Storage: 썸네일과 완성 영상만 보관
- 로컬 Python Worker: 대기 작업을 선점하고 기존 영상/TTS/게시 스크립트 실행
- 로컬 디스크: 원본 및 중간 영상 파일 보관

PC가 꺼져 있으면 작업은 `queued`로 남는다. Worker가 켜지면 한 작업을 원자적으로 선점해 실행하고 진행률, 로그, 결과를 Supabase에 기록한다.

## 승인 경계

- 콘텐츠 생성은 원격으로 요청할 수 있다.
- 완성 영상은 관리자에서 검수하고 수정하거나 게시 승인한다.
- Instagram 게시와 광고 집행에는 각각 명시적인 사용자 승인이 필요하다.
- 게시와 광고는 자동 재시도하지 않는다.
- 중복 클릭이나 Worker 재연결로 같은 게시물이 중복 게시되지 않도록 멱등 키를 둔다.

## 데이터 모델

- `products`: 상품명, 쿠팡/AliExpress URL, 파트너스 링크, 가격, 썸네일, 현재 단계, 활성 여부
- `jobs`: 작업 종류, 상태, 진행률, Worker, 잠금 만료, 시도 횟수, 오류 요약, 멱등 키
- `content`: 대본, Instagram 캡션, 해시태그, TTS 엔진·음성·속도와 버전
- `assets`: 자산 종류, 로컬 경로, Storage 경로, 형식, 크기, 영상 길이, 검수 상태
- `publications`: 게시 승인, 승인 시각, Instagram 미디어 ID·URL, 게시 시각, 실패 정보
- `campaigns`, `metrics`: A/B 광고안, 예산, 승인, Meta 식별자와 날짜별 성과
- `activity_logs`, `workers`: 중요 동작과 Worker 온라인 상태, heartbeat, 버전

## 화면 구조

- 대시보드: Worker 상태, 대기·실행·실패 작업, 현재 상품과 파이프라인
- 상품: 목록, 검색, 필터, 신규 등록
- 상품 상세: 개요, 자료, 대본/캡션, 완성 영상, 실행 이력 탭
- 작업: 진행률, 로그, 실패 원인, 안전한 재시도
- 게시: 영상 검수, 문구 수정, Instagram 게시 승인과 결과
- 광고: A/B 소재, 승인, 집행 상태와 성과 비교
- 설정: 연동 상태와 Worker 정보. 비밀키 값은 표시하지 않는다.
- 활동 기록: 요청, 승인, 실패, 재시도 이력

## 비주얼 시스템

Open Design의 `example-open-design-landing`을 참고한 Atelier Zero 편집 언어를 사용한다. Monocle, Apartamento, Études 계열의 잡지 편집감은 유지하되 관리 도구의 명료성을 우선한다.

- 크림색 종이 배경, 잉크 블랙, 버밀리언 오렌지 포인트
- 큰 세리프 헤드라인과 작은 산세리프 운영 정보
- 비대칭 그리드, 얇은 규칙선, 각진 인쇄 라벨
- 상품 이미지와 영상 프레임을 활용한 절제된 콜라주
- `ISSUE 001`, `OBJECT 001` 형태의 상품 번호
- Headroom 형태의 상단 내비게이션과 절제된 스크롤 리빌
- 모바일은 단일 열과 하단 내비게이션
- 로그, 표, 위험 작업에는 장식을 줄이고 정보 명확성을 우선
- 상태는 색상뿐 아니라 아이콘과 텍스트 라벨로 함께 표시

## 보안

- Supabase 회원가입은 닫고 허용된 이메일 한 개만 접근한다.
- 모든 비공개 테이블과 Storage 버킷에 RLS를 적용한다.
- 브라우저에는 공개 anon 키만 둔다.
- Supabase service role, Typecast, Meta 비밀키는 로컬 Worker의 `.env`에만 둔다.
- 완성 영상은 private bucket에 저장하고 만료되는 signed URL로 재생한다.
- 공개 링크 페이지 데이터와 관리자 데이터를 분리한다.
- 게시 및 광고 승인 이력은 append-only 활동 기록에 남긴다.

## 오류 처리

- Worker heartbeat가 끊기고 잠금이 만료되면 작업을 실패 또는 재개 가능한 대기로 복구한다.
- 실패 화면에는 사용자용 요약과 진단 로그를 분리한다.
- 콘텐츠 생성 작업은 제한된 횟수로 재시도할 수 있다.
- 외부 부작용이 있는 Instagram 게시와 광고는 자동 재시도하지 않는다.
- Storage 업로드 완료 전에는 자산을 사용 가능 상태로 표시하지 않는다.

## 기존 시스템 이행

- `ops/pipeline.json`은 초기 데이터를 Supabase로 옮기는 입력으로 사용한다.
- 기존 Python 스크립트는 즉시 재작성하지 않고 Worker 어댑터가 호출한다.
- Claude가 새 스크립트를 추가하면 작업 핸들러를 등록해 관리자에 연결한다.
- 기존 공개 `site/` 배포와 상품 링크 페이지는 계속 동작해야 한다.

## 검증 기준

1. 허용되지 않은 사용자는 로그인하거나 데이터를 읽을 수 없다.
2. PC가 꺼져 있는 동안 요청한 작업이 보존된다.
3. Worker가 같은 작업을 중복 실행하지 않는다.
4. 진행률과 로그가 관리자에 갱신된다.
5. 실패 작업을 안전하게 재시도할 수 있다.
6. 완성 영상을 모바일에서 재생하고 검수할 수 있다.
7. 승인 없이 Instagram 게시나 광고 집행이 불가능하다.
8. 중복 요청이 같은 릴스를 두 번 게시하지 않는다.
9. 모바일과 데스크톱에서 핵심 기능을 사용할 수 있다.
10. 기존 공개 링크 페이지와 Netlify 배포가 유지된다.

## 단계적 범위

### 1차

로그인, 상품 CRUD, 대시보드, 작업 큐, Worker heartbeat, 기존 스크립트 실행, 진행률/로그, 완성 영상 업로드와 검수까지 구현한다.

### 2차

Instagram 게시 승인과 업로드, 게시 결과를 연결한다.

### 3차

광고 A/B 승인, 집행, 성과 수집과 비교 분석을 연결한다.
