# 오늘의신기템 성과 중심 관리자 콘솔 설계

날짜: 2026-07-12  
상태: 사용자 승인됨

## 배경

현재 온라인 관리자는 인증과 Supabase 데이터 조회까지는 동작하지만, 큰 편집형 히어로와 장식 요소가 화면 대부분을 차지한다. 내비게이션은 화면을 실제로 전환하지 않고, 신규 상품 등록과 일부 더빙 요청을 제외한 버튼은 동작하지 않는다. 그 결과 “어떤 상품이 게시되었는지, 무엇이 잘 팔리는지, 다음에 무엇을 해야 하는지”를 판단하기 어렵다.

이번 개편은 Open Design의 편집 감각을 브랜드 포인트로만 남기고, 정보 밀도와 작업 실행성이 높은 현대적인 운영 콘솔로 전환한다.

## 목표

관리자가 한 화면에서 다음 질문에 답할 수 있어야 한다.

1. 현재 어떤 상품과 릴스가 운영 중인가?
2. 어느 상품이 클릭, 주문, 매출, 수수료 성과가 좋은가?
3. 광고비 대비 수익이 좋은 상품은 무엇인가?
4. 현재 대기·실행·실패한 자동화 작업은 무엇인가?
5. 외부 데이터 연결이 안 된 이유와 다음 연결 단계는 무엇인가?
6. 상품별로 다음에 실행할 안전한 작업은 무엇인가?

## 비목표

- 승인 전인 쿠팡 파트너스 Reporting API 수치를 추정하거나 가짜 데이터로 채우지 않는다.
- 공개 링크 허브의 `products.json` 스키마를 변경하지 않는다.
- 이 단계에서 Meta 광고 생성·집행 API를 완성하지 않는다.
- 이 단계에서 GA4, 쿠팡, Meta 비밀키를 브라우저에 저장하지 않는다.
- 관리자 화면을 마케팅 랜딩 페이지나 콘텐츠 포트폴리오처럼 구성하지 않는다.

## 설계 원칙

### 사실만 보여준다

- Supabase에서 읽은 실제 상품, 작업, Worker 데이터만 숫자로 표시한다.
- 아직 연결되지 않은 GA4, 쿠팡, Meta 지표는 `0`이 아니라 `연결 대기`로 표시한다.
- 마지막 동기화 시각과 데이터 출처를 함께 표시한다.
- 데이터가 오래되면 정상 값처럼 보이지 않도록 `업데이트 필요` 상태를 표시한다.

### 모든 조작은 의미가 있어야 한다

- 클릭 가능한 요소는 실제 화면 전환, 상세 열기, 새로고침 또는 기존 Supabase 명령 실행 중 하나를 수행한다.
- 아직 구현되지 않은 동작은 비활성 버튼으로 표시하고, 바로 옆이나 툴팁에 이유와 선행 조건을 쓴다.
- Instagram 게시와 광고 집행은 연결 후에도 별도 확인 절차 없이는 실행하지 않는다.

### 판단 속도를 우선한다

- 첫 화면에서 운영 상태, 성과, 오류를 스크롤 없이 최대한 파악한다.
- 표는 비교 가능한 동일 열 구조를 사용하고 정렬·검색·필터를 제공한다.
- 색은 상태를 보조하며, 텍스트와 아이콘으로도 같은 의미를 전달한다.

## 정보 구조

### 전역 셸

왼쪽 고정 사이드바와 상단 도구막대를 사용한다.

사이드바 메뉴:

- `개요`: 핵심 지표, 상품 성과, 최근 작업
- `상품`: 상품 전체 목록과 상세 관리
- `작업 큐`: 자동화 작업 상태와 재시도·취소
- `성과 분석`: GA4·쿠팡 성과 연결 및 분석
- `광고`: Meta 광고 연결 상태와 향후 캠페인 관리
- `연동·설정`: Worker와 외부 서비스 연결 상태

상단 도구막대:

- 현재 화면 제목과 설명
- 기간 선택기(데이터 연결 전에는 비활성 사유 표시)
- 전체 검색
- 마지막 새로고침 시각과 새로고침 버튼
- `새 상품` 주요 동작
- 관리자 계정 메뉴와 로그아웃

URL 해시와 현재 화면을 동기화해 새로고침이나 뒤로가기를 해도 위치가 유지되게 한다.

## 화면 설계

### 개요

첫 줄의 운영 카드:

- 활성 상품 수
- 대기 작업 수
- 실행 작업 수
- 실패 작업 수
- Worker 온라인 여부

두 번째 줄의 성과 카드:

- 링크 클릭
- 쿠팡 주문
- 매출
- 파트너스 수수료
- 광고비
- ROAS

외부 지표가 미연결이면 카드 안에 `연결 대기`, 필요한 서비스와 설정 화면 링크를 표시한다.

주요 본문:

- 상품 성과 표: 상품, 단계, 클릭, 주문, 전환율, 매출, 광고비, ROAS, 상태
- 최근 작업 큐: 작업 유형, 상품, 상태, 진행률, 수정 시각, 오류 요약
- 주의 필요 영역: 실패 작업, 오프라인 Worker, 누락된 파트너스 링크

### 상품

기본 표 열:

- 상품 번호·이름·썸네일
- 파이프라인 단계
- 판매가
- 파트너스 링크 상태
- 링크 클릭
- 주문
- 전환율
- 매출
- 광고비
- ROAS
- 활성 상태

검색, 단계 필터, 활성 상태 필터를 제공한다. 외부 지표는 연결되기 전까지 각 셀에 `—`와 `연결 대기` 설명을 사용한다.

행을 누르면 오른쪽 상세 패널을 연다. 패널에는 다음을 포함한다.

- 상품 기본 정보와 쿠팡·AliExpress·파트너스 링크
- 현재 단계와 다음 권장 작업
- 대본, 음성, 자막, 완성 영상, 캡션 자산 상태
- Instagram 게시 정보
- 광고 집행 정보
- 관련 작업과 오류 이력
- 현재 가능한 작업 버튼

### 작업 큐

상태 요약과 필터 뒤에 작업 표를 표시한다.

- 작업 유형
- 연결 상품
- 상태
- 진행률
- Worker
- 시도 횟수
- 생성·수정 시각
- 오류 요약
- 가능한 작업

초기 기능:

- `queued` 작업 취소
- `failed` 또는 `cancelled` 작업을 새 멱등 키로 재요청
- 상품의 더빙 작업 요청
- 파이프라인 동기화 작업 요청
- 실패 이유와 payload/result 확인

외부 부작용이 있는 게시·광고 작업은 일반 자동 재시도 대상에서 제외한다.

### 성과 분석

연동 전에는 텅 빈 차트 대신 서비스별 연결 안내를 보여준다.

- GA4: 링크 클릭과 유입 추이. Measurement ID만으로는 조회할 수 없음을 알리고 Property ID와 서버 측 자격 증명 연결 상태를 표시한다.
- 쿠팡 파트너스: 주문, 매출, 수수료, 전환율. 최종 승인과 Reporting API 키가 필요함을 표시한다.
- 통합 분석: 두 데이터 원천이 모두 준비된 뒤 상품별 클릭→주문 전환 퍼널을 활성화한다.

GA4는 외부 쿠팡 사이트의 구매를 직접 관측할 수 없으므로 클릭은 GA4, 주문·매출·수수료는 쿠팡 Reporting API를 진실 소스로 사용한다.

### 광고

Meta 연동 전에는 연결 상태와 계획된 기능을 명확히 구분한다.

- Instagram/Meta 계정 연결 상태
- A/B 광고안 목록
- 예산과 기간
- 광고비, 노출, 클릭, CTR, CPC, 전환 추정, ROAS
- 두 광고안 비교와 승자 기록

광고 생성·집행은 구현 및 권한 검증 전까지 비활성이다. 이후에도 `검토 → 확인 대화상자 → 실행 → 활동 기록` 순서를 강제한다.

### 연동·설정

서비스별 카드로 상태를 표시한다.

- Supabase: 연결 여부, 최근 조회 시각
- 로컬 Worker: 온라인/오프라인, heartbeat, 버전
- GA4 Data API: 미설정/연결/오류
- 쿠팡 파트너스 API: 승인 대기/키 미설정/연결/오류
- Meta Graph·Marketing API: 연결 여부와 권한 상태
- Typecast: 브라우저에 키를 노출하지 않고 Worker 확인 상태만 표시

비밀키의 실제 값은 어떠한 화면에도 표시하지 않는다.

## 데이터 모델과 단계별 연결

### 1차: 현재 데이터로 동작하는 콘솔

현재 Supabase 테이블을 그대로 사용한다.

- `products`: 상품 목록, 단계, URL, 가격, 활성 여부
- `jobs`: 작업 상태, 진행률, 오류, payload/result, 시도 횟수
- `workers`: 온라인 상태, heartbeat, 버전
- `assets`: 상품별 작업 산출물 상태
- `activity_logs`: 주요 조작 이력

성과 필드는 nullable 값으로 취급하며 연결 전에는 계산하지 않는다.

### 2차: 성과 수집 저장소

외부 API 승인과 자격 증명이 준비되면 서버 측 수집 작업과 일별 집계 테이블을 추가한다.

- 상품별 GA4 링크 클릭 일별 집계
- 쿠팡 상품별 주문·매출·수수료 일별 집계
- Meta 캠페인·광고안·일별 지표
- 수집 실행 상태와 마지막 성공 시각

브라우저는 외부 API를 직접 호출하지 않고 Supabase에 저장된 정규화된 집계만 읽는다.

## 상태와 오류 표현

외부 연동 상태는 다음 네 종류로 통일한다.

- `연결됨`: 최근 동기화가 정상
- `연결 대기`: 승인·키·설정이 아직 없음
- `업데이트 필요`: 마지막 성공 이후 허용 시간이 지남
- `오류`: 최근 동기화 실패, 요약과 재시도 안내 제공

작업 상태는 `대기`, `실행 중`, `완료`, `실패`, `취소`로 표시한다. 진행률은 실행 중일 때만 주 시각 요소로 사용한다.

## 시각 시스템

운영 화면의 기본은 중립적인 밝은 회색과 흰색 표면, 짙은 슬레이트 텍스트다. 오늘의신기템의 버밀리언 오렌지는 선택 상태, 주요 버튼, 작은 브랜드 라벨에만 사용한다.

- 본문과 수치는 가독성이 높은 산세리프 사용
- 큰 세리프 문구, 콜라주, 비대칭 랜딩 레이아웃 제거
- 4·8px 기반 간격, 12px 안팎의 절제된 모서리, 얕은 경계선
- KPI 숫자는 크되 카드 높이는 작게 유지
- 표 헤더 고정, 행 hover·focus·선택 상태 구분
- 상태 색상은 성공 초록, 경고 호박, 오류 빨강, 중립 회색으로 제한
- 아이콘은 장식이 아니라 탐색과 상태 설명에만 사용

Open Design의 영향은 로고 옆 작은 이슈 번호, 오렌지 라벨, 미세한 타이포 대비처럼 브랜드를 기억시키는 수준으로만 남긴다.

## 반응형과 접근성

- 데스크톱에서는 사이드바와 데이터 표를 사용한다.
- 좁은 화면에서는 사이드바를 오버레이 메뉴로 바꾸고 표를 핵심 필드 카드로 전환한다.
- 상품 상세 패널은 데스크톱 오른쪽 drawer, 모바일 전체 화면 sheet로 표시한다.
- 모든 조작은 키보드로 접근할 수 있고 명확한 focus ring을 제공한다.
- 아이콘 단독 버튼에는 접근 가능한 이름을 제공한다.
- 색상만으로 상태를 전달하지 않는다.
- 동작 중에는 중복 클릭을 막고 로딩 상태를 텍스트로 알린다.

## 안전 경계

- 브라우저에는 Supabase anon 키만 둔다.
- RLS가 모든 관리자 데이터 읽기·쓰기를 제한한다.
- service role, 쿠팡 secret, Meta token, Typecast key는 서버 또는 Worker에만 둔다.
- 게시와 광고 집행은 명시적 확인과 활동 기록이 필수다.
- 재시도는 새 작업을 만들며 기존 실패 기록을 덮어쓰지 않는다.

## 검증 기준

1. 모든 사이드바 메뉴가 서로 다른 실제 화면을 연다.
2. 새로고침과 뒤로가기를 해도 현재 화면이 유지된다.
3. 실제 Supabase 상품·작업·Worker 데이터가 올바르게 렌더링된다.
4. 미연결 성과 수치는 `0`으로 오해되지 않고 `연결 대기`로 표시된다.
5. 상품 검색과 상태 필터가 동작한다.
6. 상품 행을 누르면 관련 링크·자산·작업을 포함한 상세 패널이 열린다.
7. 작업 취소와 안전한 재시도는 허용된 상태에서만 가능하다.
8. 구현되지 않은 버튼은 비활성 상태와 이유를 제공한다.
9. Worker 오프라인과 실패 작업이 첫 화면에서 눈에 띈다.
10. GitHub 로그인, 로그아웃과 관리자 RLS가 유지된다.
11. 데스크톱과 모바일에서 핵심 조회·작업이 가능하다.
12. 공개 링크 허브와 기존 자동화 스크립트가 영향을 받지 않는다.

## 구현 순서

1. 전역 셸과 실제 화면 전환
2. 현재 Supabase 데이터 모델 확장과 파생 요약
3. 개요·상품·작업 큐의 실제 기능
4. 상품 상세 패널과 안전한 작업 액션
5. 성과·광고·설정의 연결 상태 화면
6. 반응형·접근성·오류·빈 상태 정리
7. 외부 API 승인 후 성과 수집 백엔드 추가
